# Seeker dApp Store Strategy

**Project:** Cypherpunk Code (cypherpunk-code.ca)  
**Constraint:** 100% free for users — monetize from other channels  
**Last updated:** June 2026

---

## Executive summary

The **Solana Seeker dApp Store** is a valuable **secondary distribution channel** for Cypherpunk Code — but not the primary launch strategy.

**Recommended approach:**

```
Web-first (PWA)  →  Build broad audience  →  Add Seeker dApp as bonus channel
```

For a 100% free crypto education project, maximum reach comes from web + social (X, YouTube, Nostr). The Seeker dApp becomes powerful once the web platform has traction and interactive onchain features are ready.

---

## Why not dApp-first?

### Pros of Seeker dApp Store

| Advantage | Detail |
| --- | --- |
| Engaged audience | Seeker owners are already deep into crypto and privacy |
| Onchain education | Wallet connect, simulations, verifiable certificates |
| Ecosystem visibility | Seeker Spotlight and rewards programs |
| Value alignment | Cypherpunk/privacy values match Seeker audience |
| Zero platform fees | 0% fees, growing ecosystem (1,000+ apps, 150k+ devices) |

### Cons of dApp-first

| Risk | Detail |
| --- | --- |
| Smaller audience | Seeker base < general web/YouTube reach |
| Development cost | Full Android dApp adds cost for limited education reach |
| Retention challenge | dApp stores favor gamification over pure education |
| Monetization limits | Project must stay 100% free on-platform |
| Accessibility | Web reaches everyone; dApp requires specific hardware |

**Verdict:** Build the web platform first. Launch Seeker dApp when interactive onchain features justify the investment.

| Feature | Why |
| --- | --- |
| **Seed Vault integration** | Biometric-secured key management for onchain exercises |
| **Mobile Wallet Adapter** | Seamless wallet connect for quizzes and certificates |
| **Privacy-themed dark UI** | Matches cypherpunk brand and Seeker aesthetic |
| **Offline-first PWA core** | Education content cached locally |
| **Onchain progress verification** | Soulbound certificates prove completion |
| **Performance optimization** | Fast load times on Seeker hardware |

---

## Decision framework

Launch the Seeker dApp when these conditions are met:

- [ ] Web platform has 1,000+ monthly active learners
- [ ] Interactive wallet features are stable on web
- [ ] Onchain certificate system is tested and working
- [ ] At least 2 learning paths have onchain quiz integration
- [ ] Community is actively requesting mobile-native experience
- [ ] Grant or sponsorship funding covers dApp development cost
