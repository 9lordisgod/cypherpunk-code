# How to Study

The most common mistake in crypto education is starting at the top — wallets, exchanges, price charts — without understanding **why** any of it matters. Cypherpunk Code is designed to be studied **from the bottom up**: foundations first, tools second, advanced sovereignty last.

Think of learning like building a tower. You cannot place the roof before the foundation.

## Daily study habits that work

### The 30-minute rule

Study **30 focused minutes per day** rather than 4-hour weekend binges. Consistency beats intensity for technical material.

### Active learning, not passive scrolling

| Passive (avoid) | Active (do this) |
| --- | --- |
| Watching videos without notes | Summarize each section in your own words |
| Bookmarking without reading | Complete one resource before adding another |
| Jumping between topics | Follow one learning path to completion |
| Reading about nodes | Actually run a testnet or regtest node |

### Keep a study log

Track what you read, what you practiced, and what confused you. A simple text file is enough. Review weekly.

### Teach what you learn

Explain a concept to someone (or write it down as if teaching). If you cannot explain it simply, you do not understand it yet.
